# 开发工作流

## 每次开发前

- 阅读 `CLAUDE.md`。
- 查看 `docs/development-plan.md` 当前阶段。
- 查看当天开发日志，如果不存在则创建。
- 明确本次只推进一个小目标。

## 每次开发中

- 先读相关文件，再修改代码。
- 修改前说明将要改动的范围。
- 优先沿用已有结构和命名。
- 避免无关重构。
- 重要编辑后运行对应检查。

## 每次开发后

- 更新当天开发日志。
- 记录完成事项。
- 记录遗留问题和下一步待办。
- 如果运行了命令，记录关键验证结果。

## 日志路径

每日开发日志位于：

```text
dev-logs/daily/YYYY-MM-DD.md
```

日志模板位于：

```text
dev-logs/daily-log-template.md
```

## 推荐日志格式

```md
# YYYY-MM-DD 开发日志

## 今日目标

## 已完成

## 验证结果

## 待办

## 风险与备注
```

## 验证策略

- 结构文件修改：检查文件是否存在、路径是否正确。
- React 代码修改：运行 TypeScript 检查或构建。
- 前端交互修改：启动本地服务并使用浏览器验证关键路径。
- 视觉修改：检查桌面视口截图，确认没有明显遮挡和错位。
