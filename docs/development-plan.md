# 分阶段开发计划

## 推进原则

- 每次只完成一个小阶段。
- 每个阶段结束后运行可用性检查。
- 不在同一步同时做大量 UI、状态和复杂交互。
- 每日更新开发日志，记录完成事项与待办事项。

## 阶段 0：项目基础

目标：建立可运行的 React + TypeScript 项目。

事项：

- 创建 Vite React TypeScript 项目。
- 安装 Konva、react-konva、lucide-react。
- 建立基础目录结构。
- 创建暗色编辑器基础布局。

验收：

- `npm run dev` 可启动。
- 页面出现顶部、左侧、中间、右侧、底部区域。

## 阶段 1：素材上传与素材栏

目标：用户可上传 PNG 并看到缩略图。

事项：

- 实现 PNG 文件选择。
- 使用 FileReader 生成 Data URL。
- 过滤非 PNG 文件。
- 左侧素材栏展示缩略图。

验收：

- 上传 PNG 后缩略图可见。
- 透明背景素材缩略图显示正常。

## 阶段 2：画布放置与基础选择

目标：素材可以添加到画布并被选中。

事项：

- 创建 Konva Stage。
- 点击素材添加画布元素。
- 画布渲染图片。
- 点击元素设置选中状态。

验收：

- 素材可出现在画布中央。
- 选中元素出现 Transformer。

## 阶段 3：元素编辑

目标：元素可拖动、缩放、旋转、透明度调整。

事项：

- 实现拖动位置更新。
- 实现缩放尺寸更新。
- 实现旋转角度更新。
- 属性面板同步显示和修改。

验收：

- 鼠标操作和属性面板操作状态一致。
- 刷新前编辑状态在当前会话内稳定。

## 阶段 4：图层管理

目标：支持背景层、中景层、前景层。

事项：

- 元素增加 layer 和 zIndex。
- 右侧按三层展示元素。
- 支持切换元素层级。
- 支持上移一层、下移一层。

验收：

- 图层变更后遮挡关系正确。
- 图层面板选中项与画布一致。

## 阶段 5：复制、删除、多选基础

目标：完成常用编辑动作。

事项：

- 实现复制当前元素。
- 实现删除当前元素。
- 支持 Shift 多选。
- 多选时支持统一删除和复制。

验收：

- 单选和多选行为清晰。
- 删除后选中状态不会残留错误 ID。

## 阶段 6：导出 PNG 与背景模式

目标：导出最终画面。

事项：

- 支持透明、白色、自定义背景。
- 使用 Konva 导出 PNG。
- 确保 UI 控件不进入导出图。

验收：

- 导出 PNG 与画布可见结果一致。
- 透明背景模式导出透明 PNG。

## 阶段 7：整理与首版验收

目标：完善第一版稳定性。

事项：

- 处理空状态。
- 补充基础错误提示。
- 检查桌面布局。
- 整理代码结构。

验收：

- 从上传到导出完整流程可走通。
- 无明显控制台错误。
